# Grill Me（需求烧烤）

**Matt Pocock 最受欢迎的技能！34.9k stars 项目核心。**

## 核心理念

> "没人确切知道自己想要什么" —— 《程序员修炼之道》

**最常见的失败模式**：误解。你认为开发者理解了你的需求，但看到他们构建的东西——才发现完全不对。

AI时代同样如此。你和AI之间存在沟通鸿沟。解决方案是**烧烤会话**——让AI反复追问你要构建的细节。

## 硬性规则

```
⚠️ 在达到共同理解之前：
   - 禁止写任何代码
   - 禁止采取任何实现行动
   
必须逐个解决决策树的每个分支。
```

## 流程

一次问一个问题，等待用户反馈后再继续。

如果问题可以通过探索代码库回答，那就先探索代码库。

### 对于每个问题

1. 提供你的推荐答案
2. 等待用户反馈
3. 根据反馈调整
4. 继续下一个问题

### 追问方向

- **目的**：为什么做这个？
- **约束**：有什么限制？
- **成功标准**：怎么算完成？
- **边界情况**：什么情况下应该/不应该工作？
- **依赖**：这个决策影响什么？
- **替代方案**：有没有其他方案？

### 挑战模糊语言

当用户使用模糊或重载的术语时，提出精确的规范术语：
- "你说'账户'——是指 Customer 还是 User？这两者不同。"

### 用具体场景讨论

当讨论领域关系时，用具体场景压力测试：
- 发明探查边缘情况的场景
- 迫使用户精确概念之间的边界

## 示例对话

```
User: "我想给应用加个搜索功能"

Assistant: Q1: 搜索什么？是全站搜索还是特定模块（订单/产品/用户）？
           我的推荐：先做产品搜索，因为这是最常用的。

User: "产品搜索吧"

Assistant: Q2: 用户怎么输入搜索词？关键词？分类筛选？
           我的推荐：关键词 + 分类筛选，因为用户通常知道自己要找什么类别。

...

（继续直到决策树清晰）
```

## 何时停止

当你能：
1. 准确描述要构建什么
2. 列出所有边界情况
3. 明确成功标准
4. 识别需要的技术决策

然后转向 implementation 或 TDD。
